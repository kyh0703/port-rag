# 로드맵

## 현재 트랙

- 활성 버전: v1
- 종료 기준:
  - PDF/텍스트·마크다운/Office 문서 업로드가 청크+벡터로 pgvector에 저장되고
    문서 상태(processing/ready/failed)가 조회된다.
  - gRPC Search가 사용자 스코프 필터된 top-k 청크를 p95 < 200ms로 반환한다.
  - voice-agent가 실제 대화 턴에서 검색 결과를 컨텍스트로 주입받는 스모크가
    확인된다.

## 향후 버전

- `v2`:
  - 목표: 검색 품질 고도화 — 하이브리드(BM25 + 벡터) 검색, 리랭커 도입
  - 종속성: v1 인제스트 파이프라인과 gRPC Search 계약 안정화
- `v3` (후보):
  - 목표: 처리량 확장 — 인제스트 큐 분리(docling-serve 또는 워커 풀),
    OCR/이미지 문서 지원
  - 종속성: v1 운영 지표로 병목 확인

## 지연된

- 후보 버전: 대화 스코프 문서(특정 대화 전용 지식), MCP tool 노출,
  실시간 인제스트 진행률 알림
- 공개 순서 질문: 하이브리드 검색과 처리량 확장 중 무엇이 먼저 필요한지는
  v1 실사용 검색 품질/문서량 데이터로 결정
